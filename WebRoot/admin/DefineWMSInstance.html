This is an app to create a WMS instance settings file for RIMapperWMS.

RIMapperWMS can be installed as multiple WMS 'instances': Each instance will be created by installing a copy of the RIMapperWMS servlets as a web application in your servlet container. In most Java servlet containers, this would be a seperate folder in the webapps directory. In each webapp a wms_instance.xml file defines the DB connection this instance will use.

As an example, when the demo from the download is installed in the TomCat directory webapps/RIMapper/, the wms_instance.xml in that directory will use PostGIS as its Spatial Feature Server. It will connect to a DB 'rimapper' (created by the load script included in the download), as user 'rimapper' with password 'rimapper'. When changing or adding DB priviliges and/or users for the RIMapper DB, you'll have to change these settings accordingly!

For now, we only include this small utility to generate the xml file client-side. We do NOT support writing the wms_instance.xml file server-side, for obvious security reasons. Just generate the XML, then copy and paste into the actual wms_instance.xml file (in the WebRoot folder)...

Spatial Feature Server:
Database: = name of database
Username: = database user for instance (advice: do not use root un/pw!)
Password: = password for user (enter in plain text, will be encrypted in file)
